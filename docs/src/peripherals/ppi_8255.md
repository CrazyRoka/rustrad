### Intel 8255 PPI Controller

The Intel 8255 PPI manages parallel I/O operations. It interfaces directly with the AY-3-8912 PSG, keyboard matrix, cassette recorder, CRTC VSYNC, and the printer port.

#### Hardware Revisions & Part Numbers
The Amstrad CPC utilised several almost identical versions of the 8255 PPI throughout its production lifetime:
* **Mitsubishi M5L8255AP-5**
* **NEC D8255AC-2** / **D8255AC-5**
* **Toshiba TMP8255AP-5**
* **KP580BB55A** (A Soviet clone of the i8255, populated on the mainboards of the East German KC Compact and the Aleste 520EX clones).

*Emulator Note:* These chips are functionally identical under standard operating modes, but they can be distinguished by writing to and subsequently reading from the write-only control register, which returns different floating or chip-specific bit patterns depending on the manufacturer.

#### I/O Port Address Decoding
The PPI is selected when bit 11 of the active I/O address is driven low (`A11 = 0`). Bits 9 and 8 select the target PPI register or control port. To prevent conflicts with other system hardware, address lines `A15`–`A12` and `A10` should be driven high (`1`), while `A7`–`A0` are ignored:

| Address Bit 9 (A9 / A1) | Address Bit 8 (A8 / A0) | PPI Selected Register | Access Type | Recommended Port |
| :---: | :---: | :--- | :--- | :--- |
| 0 | 0 | Port A Data | Read / Write | `&F4xx` |
| 0 | 1 | Port B Data | Read / Write | `&F5xx` |
| 1 | 0 | Port C Data | Read / Write | `&F6xx` |
| 1 | 1 | Control Register | Write Only | `&F7xx` |

### PPI Port Configurations

##### Port A (Bi-Directional PSG Bus)
* **OS Default State:** Mode 0, configured as Output.
* **Operation:** 
  * Port A must be dynamically set to **Output** when writing registers or select addresses to the PSG.
  * Port A must be set to **Input** when reading registers or I/O ports from the PSG.
  * If a port is configured as input, its outputs go to a high-impedance state, and reading it externally yields `&FF`.

##### Port B (System Status - Input Only)
* **OS Default State:** Mode 0, configured as Input.

| Bit | Signal Name | Description / Values |
| :---: | :--- | :--- |
| **7** | Cassette Read Data | Serial bitstream read from tape. |
| **6** | Parallel Port Busy | Printer status: `1` = Not Ready, `0` = Ready. |
| **5** | `/EXP` | Expansion port presence detector. Can be driven by peripherals (`1` = Connected, `0` = Not Connected). Also routes to ROM bank selection (bank 0 or 7) on the DDI-1 interface. |
| **4** | LK Jumper 4 | Screen Refresh Rate: `1` = 50 Hz, `0` = 60 Hz. (On KC Compact, this bit is fixed to `1` and ignored by the BIOS). |
| **3** | LK Jumper 3 | Distributor ID Bit 2 (Usually set to 4=Awa, 5=Schneider, or 7=Amstrad). |
| **2** | LK Jumper 2 | Distributor ID Bit 1. |
| **1** | LK Jumper 1 / `/TEST` | Distributor ID Bit 0 / KC Compact `/TEST` input line. |
| **0** | CRTC VSYNC / Fake VSYNC | Vertical Sync status: `1` = VSYNC Active, `0` = VSYNC Inactive. |

* **The Fake VSYNC Feature:** Port B and Port C output buffers feature a high-current Darlington drive capability, allowing any set of eight pins to source up to 1 mA at 1.5V. Under default operation, Port B Bit 0 acts as an input to monitor the CRTC VSYNC signal. If Port B is reprogrammed as an **Output** and a `1` is written to Bit 0, the PPI's high-current output driver physically overpowers the CRTC's VSYNC output pin (bus fighting). On mainboard revisions without protective inline series resistors, this forces a "fake VSYNC" state directly onto the shared line monitored by the Gate Array.

##### Manufacturer Jumper Mapping (Bits 3, 2, 1)
These bits are hardwired on the PCB layout (unconnected or tied to GND) and represent the brand name:

| Bit 3 (LK3) | Bit 2 (LK2) | Bit 1 (LK1) | Decoded Manufacturer Name |
| :---: | :---: | :---: | :--- |
| 0 | 0 | 0 | Isp |
| 0 | 0 | 1 | Triumph |
| 0 | 1 | 0 | Saisho |
| 0 | 1 | 1 | Solavox |
| 1 | 0 | 0 | Awa |
| 1 | 0 | 1 | Schneider |
| 1 | 1 | 0 | Orion |
| 1 | 1 | 1 | Amstrad |

##### Port C (Control Signals & Keyboard Row - Output Only)
* **OS Default State:** Mode 0. Upper and lower nibbles are both configured as Output.

| Bit | Controlled Signal | Functional Operation |
| :---: | :--- | :--- |
| **7** | PSG BDIR | Bus Direction control for PSG (see selection table below). |
| **6** | PSG BC1 | Bus Control line for PSG (see selection table below). |
| **5** | Cassette Write Data | Serial bitstream output to tape. |
| **4** | Cassette Motor | Power control: `1` = Motor On, `0` = Motor Off. |
| **3 - 0** | Keyboard Row Select | 4-bit binary value (`0` to `15`) selecting the active row to scan on the keyboard matrix. |

### PPI Control Register (`&F7xx`)

The function of the write-only Control Register varies depending on the state of **Bit 7** of the data byte written to it.

#### 1. Bit Set/Reset Mode (Bit 7 = 0)
When Bit 7 is `0`, the control register targets and modifies a single, specific bit within Port C without affecting the other pins:

```
  Bit 7    Bit 6    Bit 5    Bit 4    Bit 3    Bit 2    Bit 1    Bit 0
+--------+--------+--------+--------+--------+--------+--------+--------+
|   0    |   X    |   X    |   X    |   N2   |   N1   |   N0   |   VAL  |
+--------+--------+--------+--------+--------+--------+--------+--------+
```
* **Bit 7:** Must be `0` to select this function.
* **Bits 6–4:** Unused.
* **Bits 3–1 (N2–N0):** Specifies the targeted Port C bit index (`0` to `7`).
* **Bit 0 (VAL):** The new state to write to the specified pin (`0` = Clear, `1` = Set).

#### 2. Group Configuration Mode (Bit 7 = 1)
When Bit 7 is `1`, the written byte configures the Group Modes and I/O directions for Port A, Port B, Port C (Upper), and Port C (Lower).

```
  Bit 7    Bit 6    Bit 5    Bit 4    Bit 3    Bit 2    Bit 1    Bit 0
+--------+--------+--------+--------+--------+--------+--------+--------+
|   1    |   M1   |   M0   |  DIR-A | DIR-CU |  MODEB |  DIR-B | DIR-CL |
+--------+--------+--------+--------+--------+--------+--------+--------+
```
* **Bit 7:** Must be `1` to select this function.
* **Bits 6–5 (M1, M0):** Mode selection for Group A (Port A and Upper Port C). Always set to `00` (Mode 0) on the CPC.
* **Bit 4 (DIR-A):** Port A direction (`0` = Output, `1` = Input).
* **Bit 3 (DIR-CU):** Port C Upper bits 4–7 direction (`0` = Output, `1` = Input). Always `0` on the CPC.
* **Bit 2 (MODEB):** Mode selection for Group B (Port B and Lower Port C). Always set to `0` (Mode 0) on the CPC.
* **Bit 1 (DIR-B):** Port B direction (`0` = Output, `1` = Input). Always `1` on the CPC.
* **Bit 0 (DIR-CL):** Port C Lower bits 0–3 direction (`0` = Output, `1` = Input). Always `0` on the CPC.

> **CAUTION:** Writing a configuration byte to the PPI Control Register (with Bit 7 set to `1`) immediately resets the output latches of **Ports A, B, and C** to `00h` on physical hardware.

##### Common Configuration Bytes:
* **`&82`:** Sets Port A to Output, Port B to Input, and Port C to Output (used during standard register writing and general operations).
* **`&92`:** Sets Port A to Input, Port B to Input, and Port C to Output (used to prepare Port A to read data back from the keyboard matrix or the PSG).

#### Startup & Reset States
To properly initialize the PPI chip during a system cold start or hardware `/RESET` (which physically requires the reset pin to be held high for at least 50 μs during or after power-up), the control register is completely cleared. This forces all three ports (A, B, and C) into **Input Mode (High-Impedance State)**.

### Delta: CPC+ ASIC PPI

The Amstrad Plus ASIC replaces the physical 8255 silicon on the mainboard with internal gate logic. This integrated emulation has several implementation limitations:

* **No Group Modes:** Only Group Mode 0 is supported.
* **Hardwired Directions:** Port B is permanently locked as Input, and Port C is permanently locked as Output, regardless of any writes to the Control Register.
* **Configuration Reset Failure:** On a real 8255, configuring the ports with Bit 7 = `1` resets all port output latches to `00h`. The CPC+ ASIC fails to emulate this and leaves existing port states unchanged upon a configuration write.
