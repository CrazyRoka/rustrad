# CPU Architecture Overview

The Amstrad CPC CPU core is based on the Zilog Z80A microprocessor, driven by a 4.0 MHz master clock.

## Bus Arbitration, DMA Bandwidth, and Wait States
In the CPC architecture, the CPU does not have unhindered access to the system memory or I/O buses. The custom Gate Array chip arbitrates and synchronizes bus access between the CPU and the video generation hardware.

### Hardware-Level Memory Access Constraints
The video subsystem is designed to display continuous raster scanlines, requiring a constant flow of pixel data from RAM:
* **Video DMA Bandwidth:** The Video Gate Array reads exactly **two bytes** from RAM every **1 microsecond** (which equates to exactly 4 CPU T-states). 
* **CRTC Address Generation:** The exact memory location for this transfer is driven by addresses generated by the CRT Controller (6845).
* **CPU Throttle:** To accommodate this continuous video DMA transfer without memory conflicts, the Gate Array forces the CPU into wait states. The Z80 is restricted to a maximum of **one memory access per microsecond**.
* **Effective Speed:** Because the Gate Array automatically rounds up and aligns all memory and I/O cycles to these 1 μs (4 T-state) boundaries, the average processing speed of the 4.0 MHz Z80 is reduced to an effective rate of approximately **3.2 MHz**.

## Interrupt Architecture
The CPC relies on maskable interrupts generated by the Gate Array to handle time-critical tasks like keyboard scanning, color palette changes (raster splits), and sound queue updates.

* **Standard Vector Mode:** Typically operated in Z80 Interrupt Mode 1 (`IM 1`), which forces a jump to the fixed vector address `&0038` when an interrupt is acknowledged.
