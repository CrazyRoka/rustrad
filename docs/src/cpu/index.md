# CPU Architecture Overview

The Amstrad CPC CPU core is based on the Zilog Z80A microprocessor, driven by a 4.0 MHz master clock.

## Bus Arbitration and Wait States
In the CPC architecture, the CPU does not have unhindered access to the system buses. The custom Gate Array chip synchronizes memory and I/O access with the video generation circuitry.

* **The 1 μs (4 T-state) Constraint:** All RAM accesses are aligned to 1 μs boundaries. Because of this, the Gate Array inserts Wait states to stretch CPU instructions to the nearest 4 T-state boundary.
* **Effective Speed:** This bus contention reduces the execution rate of the Z80 from 4.0 MHz to an effective average of approximately 3.2 MHz, depending heavily on the T-state alignment of the instructions being executed.

## Interrupt Architecture
The CPC relies on maskable interrupts generated by the Gate Array to handle time-critical tasks like keyboard scanning, color palette changes (raster splits), and sound queue updates.

* **Standard Vector Mode:** Typically operated in Z80 Interrupt Mode 1 (`IM 1`), which forces a jump to the fixed vector address `&0038` when an interrupt is acknowledged.
